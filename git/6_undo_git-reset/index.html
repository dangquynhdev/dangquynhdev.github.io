git reset

git reset để undo, hủy commit cuối hoặc hủy đưa thay đổi vào vùng staging

hủy commit cuối

có thể hủy trong trường hợp commit đó chưa public (chưa đẩy lên Remote Repo bằng lệnh git push)
hai trường hợp hủy

trường hợp 1:
git reset với tham số --soft
git reset --soft HEAD~1
Trường hợp này sẽ hủy commit cuối, con trỏ HEAD sẽ chuyển về commit cha. Đồng thời những thay đổi của commit cuối được chuyển vào vùng staging nhằm để có cơ hội commit lại hoặc sửa đổi

trường hợp 2:
git reset với tham số --hard
git reset --hard HEAD~1
Khi dùng tham số --hard thì kết quả giống với dùng tham số --soft, chỉ có một khác biết là nội dung thay đổi của commit cuối không đưa đưa vào staging mà bị hủy luôn. Trường hợp này dùng khi bạn quyết định hủy hoàn toàn commit cuối

hủy git add

git reset
Nếu bạn đã dùng lệnh git add để cập nhật thay đổi vào vùng staging, bạn có thể hủy thao tác này bằng cách thực hiện lệnh

Hủy đưa một file vào staging

git reset -- filename
