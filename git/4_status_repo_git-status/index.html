git status

Sử dụng lệnh git để biết thông tin trạng thái sửa đổi, thêm mới, xóa các file trước khi thực hiện commit

Lệnh git status hiện thị thông tin khác nhau (do thêm mới, xóa đi, sửa đổi các file) giữa các file trong các trường hợp:
Khác nhau giữa các file trong vùng staging (chỉ mục) và commit tại con trỏ HEAD (Thường HEAD ở vị trí commit cuối): -> thông tin này bạn có thể thực hiện lệnh commit để lưu staging vào dữ liệu Git
Khác nhau giữa các file trong thư mục làm việc và trong staging -> bạn có thể chạy git add rồi commit
Khác nhau giữa thư mục làm việc và những file chưa được giám sát bởi Git -> bạn có thể chạy git add rồi commit

git status -s

muốn hiện thị thông tin ngắn gọn hơn thì cho thêm tham số -s

' ' = unmodified (không đổi)
M = modified (có sửa đổi)
A = added (file mới thêm)
D = deleted (file bị xóa)
R = renamed (đổi tên file)
C = copied (file copy từ file khác)
U = updated but unmerged (đã cập nhật, nhưng chưa merge)
